# /release-notes — Update Release Notes

Add an entry to `release-notes.md` documenting completed work.

## Arguments
- `$ARGUMENTS` — Description of what was completed

## Instructions
1. Read the current `release-notes.md` file.
2. Append a new entry with:
   - Date (today's date)
   - Task ID(s) completed
   - What was delivered (brief description)
   - Which agent did the work
   - Any known issues or follow-ups
3. Use the format:
```
### [Date] — [Task ID]: [Title]
- **Delivered by:** [Agent name]
- **Summary:** [What was done]
- **Known issues:** [Any issues, or "None"]
```

Update: $ARGUMENTS
